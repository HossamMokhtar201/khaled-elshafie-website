# Design System v2 — التوجه المودرن (يحل محل النسخة الكلاسيكية)

> **تعليمات لكلود كود:** هذا المستند **يستبدل بالكامل** القسم (3) "Design System" الموجود في `claude-code-master-brief.md`. كل باقي المستند (السياق، الـ Sitemap، بنية `/content/`، الـ Tech Stack، المايلستونز) **يفضل زي ما هو من غير تغيير**. المطلوب فقط إعادة تنفيذ **Milestone 2 (Design System) و Milestone 3 (تطبيقه على الصفحات) و Milestone 4 (الحركة)** بالمواصفات الجديدة هنا، فوق نفس البنية التحتية والمحتوى الجاهز من Milestone 0 و1 (يعني الـ Tokens تتغير قيمها، مش معمارية الملفات).

---

## السياق: ليه التغيير؟

النسخة الأولى كانت "كلاسيكي/رسمي جدًا" (سيريف كلاسيكي، ألوان كحلي دافئ شبيه بالورق القديم). المطلوب دلوقتي توجه **"Modern Premium" مستوحى من قوالب Framer عالية الجودة** (تمت مراجعة 4 مراجع: Clipzy, Videfo, Consult, Homemaster) - يعني: تايبوجرافي ضخم وجريء، أزرار وكروت بزوايا مدورة كبيرة (Pill/rounded-2xl)، حركة سلسة ومكونات دايناميكية (Marquee, Count-up, Carousel قابل للسحب). **مع الحفاظ على إحساس "بزنس كلاس وهيبة"** عن طريق الألوان (غامقة/محايدة، أكسنت واحد بس)، مش عن طريق الخطوط الكلاسيكية أو الزوايا الحادة زي قبل كده.

## 2) التايبوجرافي الجديد

- **العناوين (Display/Headings):** "Noto Kufi Arabic" وزن **Black/ExtraBold** (900/800) — لكن بمقاسات أكبر بكتير من النسخة القديمة، عشان يدي إحساس "Display Type" ضخم زي الريفرانسات.
- **النصوص الأساسية:** "IBM Plex Sans Arabic" Regular/Medium — نفس القديم، ده كان مناسب أصلاً.
- **الاقتباسات (بنك الاقتباسات فقط):** تم إلغاء استخدام خط "Aref Ruqaa" الكلاسيكي في كل الموقع. بدل كده، الاقتباسات بتتعرض بنفس خط العناوين (Noto Kufi Arabic) لكن بحجم كبير جدًا ولون الأكسنت، بخط سفلي (Accent underline) قصير فوق كل اقتباس — أسلوب أنضف وأكثر اتساقًا مع باقي الموقع.
- **Type Scale الجديد (أكبر من القديم، خصوصًا للعناوين الكبيرة):**

```
--text-xs: 0.8125rem   --text-sm: 0.9375rem   --text-base: 1.0625rem
--text-lg: 1.375rem    --text-xl: 1.75rem     --text-2xl: 2.25rem
--text-3xl: 3rem       --text-4xl: 4rem       --text-5xl: 5.5rem
--text-display: 7rem   /* جديد: للهيرو فقط، على الديسكتوب */
```

(على الموبايل، `--text-display` ينزل لـ `--text-4xl` تلقائيًا عبر clamp())

## 3) الأشكال والمسافات (تغيير جذري عن القديم)

- **Border Radius — كبير ومدور بدل الصغير الرسمي القديم:**

```
--radius-sm: 12px       /* بدل 4px قديمًا */
--radius-md: 20px        /* بدل 8px قديمًا */
--radius-lg: 32px        /* بدل 12px قديمًا */
--radius-pill: 9999px    /* جديد: للأزرار الأساسية كلها */
```

- **الأزرار الأساسية (Primary Buttons):** شكل **Pill كامل** (`--radius-pill`) مش مستطيل بزوايا خفيفة زي القديم.
- **الصور:** كل صورة رئيسية (هيرو، كروت البراندات، كروت المشاريع) بزاوية `--radius-lg` (32px) — مدورة بشكل واضح ومقصود، مش خفيف.
- **الظلال:** أعمق شوية من القديم عشان تدي إحساس "طفو" (Floating) للكروت:

```
--shadow-sm: 0 2px 8px rgba(10,10,11,0.06);
--shadow-md: 0 12px 32px rgba(10,10,11,0.10);
--shadow-lg: 0 24px 64px rgba(10,10,11,0.16);
--shadow-hover: 0 20px 48px rgba(212,162,78,0.18);  /* جديد: ظل بلون الأكسنت عند الـ hover */
```

## 4) مكونات جديدة مطلوب إضافتها (غير موجودة في النسخة الكلاسيكية القديمة)

هذه المكونات **إضافة** لمكتبة `/styleguide` الموجودة، ولازم تتبنى وتتستخدم في الصفحات المناسبة:

### أ) Logo/Trust Marquee

شريط أفقي بيتحرك أوتوماتيك بلا نهاية (CSS animation، لا JS ثقيل)، لعرض أي شعارات ثقة مستقبلية (شعارات وسائل إعلام ظهر فيها خالد الشافعي، أو شعارات البراندات). حاليًا بيانات placeholder (شعارات نصية بسيطة "الإعلام" حتى تتوفر شعارات حقيقية). يُستخدم أسفل الهيرو في الصفحة الرئيسية مباشرة.

### ب) Stat Counter (Count-up ضخم)

نفس فكرة الـ Stats bar القديمة، لكن التصميم الجديد: أرقام بحجم `--text-4xl` أو أكبر، بخط Black، مع سكشن خلفية غامقة (`--color-bg-dark-section`) بدل الخلفية الفاتحة القديمة — يدي تباين قوي زي الريفرانسات.

### ج) Numbered Process Section (خطوات مرقمة 01/02/03)

كومبوننت جديد: رقمين كبيرين (01, 02, 03...) + صورة كبيرة بزاوية `--radius-lg` + نص، بيتبادل الاتجاه (صورة يمين/نص شمال، ثم العكس) كل عنصر. يُستخدم في:

- صفحة "من هو خالد الشافعي" بدل الـ Timeline العمودي البسيط القديم (يتحول الـ Timeline لنفس هذا الشكل: كل محطة = خطوة مرقمة بصورة/أيقونة).
- صفحة الاستشارات: "إزاي تحجز استشارتك؟" (اختر اليوم → اختر الموعد → بياناتك → التأكيد) بنفس الشكل المرقم.
- شرح نظام اتحاد الملاك في صفحة العقارات (خطوات الآلية الست المذكورة في المحتوى) بنفس الشكل.

### د) Testimonial/Quote Carousel قابل للسحب (Draggable)

كاروسيل أفقي بيتسحب باليد (Mouse/Touch drag) أو بأزرار Prev/Next، لعرض بنك الاقتباسات في الصفحة الرئيسية (بدل الـ Ticker الثابت القديم). كل بطاقة: اقتباس كبير + خط أكسنت تحته قصير.

### هـ) FAQ Accordion

أكورديون بسيط (نُفّذ سابقًا كنص عادي في صفحة العقارات) — الآن يتحول لمكون تفاعلي فعلي (فتح/قفل بحركة سلسة) بدل عرضه كنص ثابت.

### و) Card Hover System (يُطبّق على كل الكروت في الموقع)

عند الـ hover على أي كارت (براند، مشروع عقاري، فيديو، وظيفة):

- الصورة جواه تكبر خفيف (`scale(1.04)`) بحركة سلسة.
- الكارت نفسه يرتفع (`translateY(-6px)`) والظل يتحول لـ `--shadow-hover` (بلون الأكسنت).
- لو فيه أيقونة سهم (مثلاً "اعرف أكتر ←")، تتحرك شوية في اتجاه القراءة (يمين لشمال لأننا RTL).

## 5) تحديثات الحركة (Motion) — إضافة فوق مواصفات القسم 3.5 القديمة في الـ Master Brief

- **الماركي (Logo/Quote strips):** حركة CSS خالصة (`@keyframes marquee`)، سرعة ثابتة هادئة (30-40 ثانية للدورة الكاملة)، تتوقف عند الـ hover (`animation-play-state: paused`).
- **الكاروسيل القابل للسحب:** فيزياء سحب طبيعية (Momentum/Inertia) — يُفضّل استخدام مكتبة خفيفة (زي `embla-carousel-react`) بدل بناء فيزياء السحب يدويًا، طالما متوافقة مع Static Export.
- **العدّادات (Count-up):** نفس المبدأ القديم (Intersection Observer + عد تصاعدي عند الدخول للشاشة)، لكن التصميم الجديد أكبر وأوضح.
- **جميع مبادئ القسم 3.5 القديمة (Easing، احترام `prefers-reduced-motion`، الـ Sticky Header) تفضل سارية زي ما هي.**

- مبدأ الـ Design Tokens كمصدر وحيد للحقيقة (قابل للربط بالداشبورد لاحقًا) — **بس بالقيم الجديدة دي بدل القديمة**.

## 7) خطة التنفيذ لكلود كود (مختصرة)

1. حدّث `/src/styles/tokens.css` و`/src/config/design-tokens.json` بالقيم الجديدة في هذا المستند (تجاوز القيم القديمة بالكامل).
2. أعد بناء مكتبة `/styleguide` لتشمل المكونات الجديدة (Marquee, Stat Counter الجديد, Numbered Process, Draggable Carousel, FAQ Accordion التفاعلي, Card Hover System) بجانب المكونات القديمة (الأزرار، الفورمز، إلخ) بأشكالها الجديدة (Pill/rounded-2xl).
3. أعد تطبيق التصميم الجديد على كل الصفحات بنفس ترتيب Milestone 3 الأصلي، مستبدلًا كل مكون قديم بمكافئه الجديد حيث ينطبق (خصوصًا: الهيرو الرئيسي + الماركي، صفحة "من هو" بالخطوات المرقمة، صفحة الاستشارات بخطوات الحجز المرقمة، صفحة العقارات بخطوات اتحاد الملاك المرقمة + FAQ تفاعلي، بنك الاقتباسات ككاروسيل قابل للسحب).
4. أعد تطبيق الحركة (Milestone 4) بما يشمل الإضافات الجديدة في القسم (5) أعلاه.
5. Commit وpush بعد كل مايلستون زي المتبع، بس بدون تكرار Milestone 0/1 (البنية والمحتوى ثابتين).
