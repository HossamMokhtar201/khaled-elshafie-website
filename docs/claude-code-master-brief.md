# Master Brief — موقع خالد الشافعي (Khaled ElShafie Personal Business Hub)

> **ملحوظة لأي أداة/موديل هيشتغل على المستند ده (Claude Code):** هذا المستند مصمم ليكون **self-contained** بالكامل. كل قرار بريف، محتوى، ديزاين، بنية تقنية، ومايل ستونز موجود هنا. لو في أي تفصيلة ناقصة، افترض الأنسب بناءً على باقي المستند ووثّق الافتراض في `ASSUMPTIONS.md` جوه الريبو، وكمّل الشغل من غير ما توقف لحد ما تخلص كل المايلستونز أو توصل لحاجة فعلاً بتحتاج قرار بشري (زي مفتاح API حقيقي أو نص عقد قانوني رسمي).

## 1) السياق العام للمشروع (Business Context)

الموقع ده منصة شخصية/بزنس شاملة للشيخ **خالد الشافعي** — تاجر تجزئة وخبير سوق مصري (+30 سنة خبرة)، ومستشار أعمال، صاحب براندين (المخزنجي، علي بابا)، وشريك في مشاريع عقارية بنظام "اتحاد الملاك".

المرحلة الحالية (اللي بيغطيها هذا البريف) هي: **بناء الموقع التسويقي/المحتوى الكامل (Frontend)** بكل صفحاته وسكشناته، بدون باك إند حقيقي بعد (الحجز/الدفع هيتبنى كـ UI جاهز ومتصل بـ Mock data/placeholders، على أساس تقني يسمح بربطه بباك إند حقيقي والداشبورد في مرحلة تالية من غير إعادة بناء).

### أهم الملفات المرجعية (Business Data - already collected)
كل الداتا التالية اتجمعت من مقابلتين بودكاست (بالنص الكامل)، منشورات فيسبوك، وصور شاشة حقيقية. **استخدمها كمحتوى فعلي حيث أمكن، ومكانها Placeholder واضح المعالم حيث لا تتوفر.**

راجع ملف `site-structure-blueprint.md` (مرفق سابقًا لنفس المشروع) للتفاصيل الكاملة لكل صفحة وسكشن — **هذا الملف الحالي يدمج ملخصه ويضيف عليه الديزاين والتنفيذ**. إن لم يكن الملف متاحًا في بيئة العمل، الملخص الكامل موجود في القسم 4 بالأسفل.

## 3) الـ Design System (البنية التحتية اللي هتتربط بالداشبورد لاحقًا)

### 3.1 مبدأ أساسي: Design Tokens كمصدر وحيد للحقيقة (Single Source of Truth)
**لازم** كل قيم الألوان والخطوط والمسافات تتعرّف مرة واحدة في ملف Tokens مركزي (وليس Hardcoded في الكومبوننتس)، بحيث لما نبني الداشبورد بعدين، شاشة "إعدادات الهوية البصرية" هتقرا/تكتب في نفس الملف/الجدول من غير أي إعادة بناء.

**التنفيذ التقني المقترح:**
- ملف `/src/styles/tokens.css` يعرّف كل القيم كـ CSS Custom Properties (`--color-primary-900`, `--font-heading`, `--space-4`... إلخ).
- ملف موازي `/src/config/design-tokens.json` بنفس القيم بصيغة JSON — ده اللي الداشبورد (لاحقًا) هيعدّل عليه، وعند أي تعديل يعاد توليد/حقن الـ CSS variables وقت الـ build أو عبر endpoint بسيط `/api/design-tokens` وقت الـ runtime (اختَر الأسهل تقنيًا، ووثّق القرار في `ARCHITECTURE.md`).
- Tailwind config (لو هنستخدم Tailwind، موصى به) يقرأ من نفس التوكنز عبر `theme.extend` بدل تعريف ألوان مباشرة.
- **ممنوع** أي قيمة لون/خط Hardcoded جوه أي Component — كل حاجة عن طريق التوكن.

### 3.2 لوحة الألوان (Formal / Business-Class / Classic)

هوية الموقع الرئيسية (الشخصية لخالد الشافعي) **مختلفة عن** هوية كل براند فرعي — كل براند (المخزنجي، علي بابا) بيحتفظ بألوانه الأصلية جوه صفحته، لكن الموقع الأم (Header/Footer/صفحات الاستشارات والعقارات) بيستخدم الهوية دي:

```
/* التوكنز الأساسية - Primary Palette (Deep Navy) */
--color-primary-900: #0B1826   /* أساسي غامق - خلفيات Hero وForms داكنة */
--color-primary-800: #132434
--color-primary-700: #1C3349
--color-primary-600: #2A4763   /* أساسي للأزرار/الروابط الغامقة */
--color-primary-500: #3E5F82   /* حالة Hover */

/* الأكسنت - Muted Gold/Bronze (هيبة وكلاسيكية بدون صراخ) */
--color-accent-600: #9C7A34
--color-accent-500: #B9924A   /* أساسي: تفاصيل، خطوط فاصلة، أيقونات، اقتباسات */
--color-accent-400: #D2B476   /* Hover / حالات فاتحة */

/* المحايدات - خلفية دافئة تشبه الورق الفاخر */
--color-bg: #F7F5F0
--color-bg-alt: #EFEAE0
--color-surface: #FFFFFF
--color-border: #E1DACB
--color-border-strong: #C9BFA8

/* النصوص */
--color-text-primary: #12202C
--color-text-secondary: #5B5A52
--color-text-muted: #8A8577
--color-text-inverse: #F7F5F0

/* دلالية (استخدام محدود جدًا) */
--color-success: #2F6B4F
--color-danger: #8C3B31
--color-warning: var(--color-accent-500)
```

- **لا يوجد Dark Mode في هذه المرحلة** — لكن التوكنز مبنية بطريقة تسمح بإضافته لاحقًا كطبقة `[data-theme="dark"]` بدون تعديل الكومبوننتس.
- ألوان البراندات الفرعية (المخزنجي: أحمر/أسود/أبيض/أخضر من اللوجو - علي بابا: أحمر/أصفر) تتعرّف كـ Namespace منفصل: `--brand-mokhzangy-primary`, `--brand-alibaba-primary` إلخ، وتُستخدم فقط جوه صفحات وكومبوننتس البراند المعني.

### 3.3 التايبوجرافي

- **العناوين (Headings):** خط "Noto Kufi Arabic" وزن Bold/ExtraBold — هندسي، رسمي، مهيب، بيدي إحساس "مؤسسة" لا "استارت أب".
- **النصوص الأساسية (Body):** خط "IBM Plex Sans Arabic" وزن Regular/Medium — قراءة مريحة رسمية.
- **الاقتباسات/الجمل المميزة (بنك الاقتباسات):** خط "Aref Ruqaa" أو "Amiri" (كلاسيكي/خطي) بشكل مميز داخل Cards الاقتباسات فقط، عشان يدي طابع أدبي يتماشى مع ميول خالد الشافعي الأدبية المذكورة في الترانسكريبت.
- **الأرقام/العملات/اللاتيني:** "Inter" كخط احتياطي متناسق مع الأوزان.
- كل الخطوط Google Fonts (لا تكلفة ترخيص، تدعم العربي بالكامل).
- Type Scale (Modular، أساس 16px، نسبة 1.25):
```
--text-xs: 0.75rem   --text-sm: 0.875rem   --text-base: 1rem
--text-lg: 1.25rem   --text-xl: 1.563rem   --text-2xl: 1.953rem
--text-3xl: 2.441rem --text-4xl: 3.052rem  --text-5xl: 3.815rem
```

### 3.4 المسافات والأشكال
- Spacing scale أساس 4px: `--space-1: 4px` وحتى `--space-24: 96px`.
- Border radius **صغير نسبيًا** (يدي إحساس رسمي لا "Playful"): `--radius-sm: 4px` (أزرار)، `--radius-md: 8px` (كروت)، `--radius-lg: 12px` (حاويات كبيرة/مودالز). تجنب الـ radius الكبير (20px+) في كل مكان.
- الظلال خفيفة جدًا ومحايدة (لا تستخدم ظلال ملونة/نيون):
```
--shadow-sm: 0 1px 2px rgba(18,32,44,0.06);
--shadow-md: 0 8px 24px rgba(18,32,44,0.08);
--shadow-lg: 0 16px 48px rgba(18,32,44,0.12);
```

### 3.5 الحركة (Motion) — سلسة وهادية، لا "Playful"
- Easing الأساسي: `cubic-bezier(0.22, 1, 0.36, 1)` (خروج ناعم بدون ارتداد) — يُستخدم في كل الانتقالات.
- المدد: Micro-interactions (hover/focus) = 200-250ms | انتقالات صفحة/سكشن = 500-700ms.
- **Scroll Reveal:** كل سكشن/كارت يظهر بـ fade-in + translateY(16px→0) عند الدخول للـ viewport، مرة واحدة فقط (لا تتكرر عند التمرير لأعلى ولأسفل)، باستخدام Intersection Observer (أو مكتبة زي Framer Motion `whileInView`).
- **احترام `prefers-reduced-motion`**: أي انيميشن لازم يتعطل/يتقلص تلقائيًا لو المستخدم مفعّل تقليل الحركة (accessibility إلزامي).
- **Hover States:**
  - الكروت (براندات/مشاريع/فيديوهات): ترتفع خفيف (`translateY(-4px)`) + ظل يكبر + حد رفيع بلون الأكسنت يظهر.
  - الأزرار: تعبئة تتحرك (fill sweep) من الاتجاه المناسب لـ RTL، أو تغميق تدريجي للخلفية.
  - الروابط النصية: خط سفلي (underline) يتمدد من المنتصف للخارج، لا يظهر فجأة.
  - عناصر القائمة (Nav): تحته خط رفيع بلون الأكسنت يتمدد بسلاسة.
- **Sticky Header:** يختفي عند التمرير لأسفل ويظهر فورًا عند التمرير لأعلى، مع خلفية Blur/شبه شفافة بعد تجاوز أول شاشة.
- **عدّاد الأرقام (Stats bar):** الأرقام (35 فرع، 600K+ متابع، +30 سنة خبرة) تُحسب تصاعديًا (Count-up animation) عند ظهورها لأول مرة في الشاشة.
- **Smooth Scroll:** تفعيل `scroll-behavior: smooth` للتنقل بالروابط الداخلية (Anchor links)، وفتح المجال لاحقًا لمكتبة Lenis/Locomotive لو الأداء سمح، لكن **بدون** تأثيرات Parallax صاخبة — أقصى حاجة Parallax خفيف جدًا (5-10% فرق سرعة) في خلفية الـ Hero فقط، وليس في باقي الصفحة.
- **لا** لأي انيميشن "دخول درامي" (bounce، rotate، elastic) — كل حاجة Fade/Slide هادي فقط، يتماشى مع الطابع الرسمي.

### 3.6 مكونات UI أساسية مطلوب بناؤها كـ Design System معزول (قبل صفحات المحتوى)
ابنِ صفحة داخلية `/styleguide` (مش هتُنشر للعامة، تُستخدم للمراجعة الداخلية فقط) تعرض كل الكومبوننتس دي بكل حالاتها (Default/Hover/Focus/Disabled):
- الأزرار (Primary / Secondary / Ghost / Link) بثلاث أحجام
- الكروت (Brand Card / Project Card / Video Card / Quote Card / Testimonial Card)
- الـ Navigation (Header Desktop + Mobile Drawer، Footer)
- الفورمز (Input, Textarea, Select, Date/Time Picker للحجز، File Upload للوظائف)
- Badges/Tags (لحالة المشروع العقاري: متاح/محجوز جزئيًا/مكتمل)
- Modal / Drawer (لتفاصيل سريعة)
- Tabs (تُستخدم في صفحة المحتوى: بودكاست/فيديو)
- Stats Counter component
- Quote/Testimonial carousel
- Timeline component (لصفحة السيرة الذاتية)
- Toast/Alert (تأكيد نجاح الحجز/الإرسال)

## 5) بنية المحتوى التقنية (لضمان جاهزية الداشبورد لاحقًا)

كل محتوى الموقع (نصوص، أسعار، مشاريع، اقتباسات، فروع، فيديوهات) **يُخزَّن كملفات JSON منظمة** جوه `/content/` وليس Hardcoded داخل الـ Components، حتى لو مفيش CMS حقيقي بعد. كل Component بيقرأ من الملف المقابل له. أمثلة:
```
/content/
  ├── site-settings.json      // اسم الموقع، الشعار، بيانات التواصل
  ├── quotes.json             // بنك الاقتباسات
  ├── stats.json              // أرقام الـ Stats bar
  ├── timeline.json           // محطات السيرة الذاتية
  ├── brands/
  │   ├── mokhzangy.json      // بيانات براند المخزنجي + الفروع
  │   └── alibaba.json
  ├── real-estate/
  │   ├── projects.json       // قائمة المشاريع (index)
  │   └── beit-elwatan-tg5.json  // تفاصيل مشروع كامل
  ├── content-hub/
  │   ├── podcast.json
  │   └── videos.json
  ├── consultations/
  │   ├── services.json       // مجالات وأنواع الاستشارات
  │   └── availability.json   // Mock مواعيد متاحة
  └── careers.json
```
**الفائدة:** لما نبني الداشبورد بعدين، كل شاشة فيه هتكون ببساطة CRUD على نفس ملفات الـ JSON دي (أو الجداول اللي هتتبنى منها في قاعدة بيانات لاحقًا بنفس الـ Schema بالظبط) — **صفر إعادة بناء** في الفرونت إند.

## 6) الـ Tech Stack والبنية التقنية

- **Framework:** Next.js (أحدث إصدار مستقر، App Router) + TypeScript.
- **Styling:** Tailwind CSS مربوط بالـ Design Tokens (قسم 3.1) + CSS Modules عند الحاجة لتفاصيل دقيقة.
- **Animation:** Framer Motion (لـ scroll reveal، hover states، الترانزيشن بين الصفحات).
- **الاتجاه:** RTL بالكامل (`dir="rtl"` على مستوى `<html>`)، مع دعم مكونات RTL-aware (خصوصًا الأيقونات ذات الاتجاه زي الأسهم).
- **الخطوط:** Google Fonts محمّلة عبر `next/font` للأداء الأمثل.
- **الصور:** `next/image` لكل الصور مع lazy loading وoptimizaton تلقائي.
- **إدارة الفورمز:** React Hook Form + Zod للتحقق (Validation)، حتى لو الإرسال الفعلي (API) مش مفعّل بعد — نجهز الـ UX كامل مع رسائل خطأ/نجاح واضحة.
- **إمكانية الوصول (Accessibility):** التزام بـ WCAG AA كحد أدنى (تباين ألوان كافٍ، focus states واضحة، semantic HTML، aria-labels).
- **الأداء:** استهداف Lighthouse 90+ في كل المحاور (Performance/Accessibility/Best Practices/SEO).
- **SEO الأساسي:** meta tags ديناميكية لكل صفحة، Open Graph tags، sitemap.xml، robots.txt.

### 6.1 قرار الاستضافة (مهم — يحتاج تحقق فعلي من Claude Code قبل المايلستون الأخير)
- الريبو على **GitHub**، النشر المؤقت للمراجعة على **Vercel** (preview تلقائي مع كل push).
- النشر النهائي على **Hostinger Cloud Startup**.
- ⚠️ **تنبيه هام:** خطط Hostinger Cloud تدعم استضافة Node.js لكن بحدود معينة (قد لا تدعم كل ميزات Next.js SSR/Serverless بنفس كفاءة Vercel). **قبل المايلستون الأخير، لازم Claude Code يتحقق فعليًا من:**
  1. هل خطة "Cloud Startup" تدعم تشغيل تطبيق Node.js (Next.js server) بشكل كامل، أم الأنسب نشر الموقع كـ **Static Export** (`next export` / `output: 'export'`)؟
  2. بما إن المرحلة دي (Frontend فقط بدون باك إند حقيقي) — **الأفضل تقنيًا** أن يُبنى الموقع بحيث يدعم Static Export بالكامل الآن (كل المحتوى من ملفات JSON محلية، لا API routes حقيقية بعد)، وهذا يضمن توافق كامل مع أي استضافة عادية (Hostinger شيرد/كلاود) بدون تعقيد.
  3. لما نبني الداشبورد والباك إند لاحقًا (حجز، دفع، CMS)، وقتها هيتقرر إذا هنحتاج نقل الاستضافة لخدمة تدعم Node كامل (Vercel/Railway/VPS على Hostinger) أو الإبقاء على الفرونت Static ونعمل الباك إند كـ API منفصل (Headless).
  - **القرار الافتراضي المطلوب تنفيذه الآن:** ابنِ الموقع بـ Next.js مع `output: 'export'` (Static) قدر الإمكان، ووثّق أي جزء مش هيقدر يبقى Static (زي التقويم/الحجز الديناميكي) في `ARCHITECTURE.md` مع اقتراح الحل البديل (Client-side fetching من JSON، أو API route بسيط لاحقًا).

## 7) المايلستونز (خطة التنفيذ التسلسلية)

**تعليمات عامة لكل المايلستونز:**
- نفّذ كل مايلستون بالكامل، اعمل commit وpush على GitHub بعد كل مايلستون برسالة واضحة (`feat(milestone-X): ...`)، وحدّث ملف `PROGRESS.md` في جذر الريبو بحالة كل مايلستون (✅ مكتمل / 🔄 جاري / ⏳ لم يبدأ).
- **لا تتوقف** للمراجعة بين المايلستونز — كمّل تلقائيًا للي بعده. التوقف فقط لو واجهت قرار بيحتاج إنسان فعليًا (مش موجود بديل منطقي في هذا المستند).
- أي افتراض تعمله (نص، سعر تقديري، صورة placeholder) يتسجل في `ASSUMPTIONS.md`.

### Milestone 0 — Project Setup & Infrastructure
- تهيئة مشروع Next.js + TypeScript + Tailwind.
- إعداد الـ Design Tokens (قسم 3) كملفات CSS Variables + JSON.
- إعداد بنية `/content/` (قسم 5) بكل ملفات الـ JSON المبدئية معبأة بالداتا المتاحة والـ Placeholders.
- إعداد الخطوط (next/font)، الـ RTL الأساسي، هيكل المجلدات (`/components`, `/app`, `/content`, `/lib`).
- ربط الريبو بـ GitHub + Vercel (preview تلقائي).
- **Definition of Done:** مشروع فاضي شغال محليًا وعلى Vercel، بيعرض صفحة index بسيطة تؤكد إن التوكنز والخطوط شغالة.

### Milestone 1 — UX Structure (الهيكل والمحتوى بدون تصميم نهائي)
- بناء كل الصفحات المذكورة في قسم 4 بـ **HTML/JSX semantic structure كامل** (headings صحيحة، landmarks، فورمز functional من ناحية الحقول)، مع كل المحتوى الفعلي/المستنبط/Placeholder موضوع في مكانه الصحيح من ملفات الـ JSON.
- لا تصميم بصري متقدم بعد (تنسيق بسيط جدًا فقط لضمان القراءة والتصفح المنطقي) — الهدف التأكد إن كل صفحة وكل سكشن ومساره منطقي وكامل قبل الدخول في التصميم البصري.
- التنقل (Routing) بين كل الصفحات شغال بالكامل.
- **Definition of Done:** يقدر أي حد يتصفح الموقع بالكامل من الرئيسية لأي صفحة فرعية ويشوف كل المحتوى المطلوب موجود، بدون أي رابط معطّل (Broken link) أو صفحة فاضية.

### Milestone 2 — Design System Implementation (UI Kit)
- تنفيذ كل الكومبوننتس الأساسية المذكورة في قسم 3.6 بالتصميم النهائي (ألوان، تايبوجرافي، مسافات، حالات hover/focus).
- بناء صفحة `/styleguide` تعرض كل الكومبوننتس بكل حالاتها.
- **Definition of Done:** صفحة الـ styleguide جاهزة ومطابقة تمامًا لمواصفات قسم 3، وكل كومبوننت فيها موثّق ومعزول.

### Milestone 3 — Page-by-Page UI Application
- تطبيق الـ Design System على كل صفحات Milestone 1 صفحة بصفحة بنفس الترتيب المذكور في قسم 4 (الرئيسية أولاً، ثم من هو، ثم البراندات، ثم الاستشارات... إلخ).
- كل صفحة بعد تطبيق التصميم لازم تُختبر بصريًا (Desktop) قبل الانتقال للتالية.
- **Definition of Done:** كل الصفحات بالتصميم النهائي الكامل على شاشة Desktop، متسقة مع بعضها البعض ومع الـ styleguide.

### Milestone 4 — Motion & Interaction Polish
- تطبيق كل مواصفات الحركة في قسم 3.5 (scroll reveal، hover states، sticky header، count-up stats، smooth scroll) على كل الصفحات.
- اختبار `prefers-reduced-motion` والتأكد من التوافق.
- **Definition of Done:** تجربة تصفح سلسة ومتسقة الحركة في كل الموقع، بدون أي انيميشن "يقفز" أو مزعج، وتعمل بشكل صحيح مع تفضيل تقليل الحركة.

### Milestone 5 — Responsive & Accessibility QA
- اختبار وتصحيح كل صفحة على: موبايل (375px)، تابلت (768px)، ديسكتوب (1440px+).
- التأكد من عمل RTL بشكل صحيح 100% (اتجاه الأيقونات، الأسهم، الـ carousels، الفورمز).
- فحص Accessibility (تباين الألوان، keyboard navigation، screen reader labels).
- **Definition of Done:** الموقع يعمل ويبدو احترافيًا على كل أحجام الشاشات، ويجتاز فحص accessibility أساسي.

### Milestone 6 — Performance, SEO & Deployment Readiness
- تحسين الأداء (Lighthouse audit وإصلاح أي نقطة ضعف).
- إضافة meta tags، Open Graph، sitemap.xml، robots.txt.
- التحقق من قرار الاستضافة (قسم 6.1) وتوثيق النتيجة والخطوات في `DEPLOYMENT.md`.
- تجهيز build نهائي (Static export أو حسب ما تقرر) وتأكيد شغله بشكل صحيح على معاينة Vercel.
- **Definition of Done:** الموقع جاهز للنشر، Lighthouse score 90+، وملف `DEPLOYMENT.md` فيه خطوات واضحة لرفعه على Hostinger.

### (مرحلة لاحقة، خارج نطاق هذا البريف): Dashboard/CMS + Backend
بعد اعتماد الموقع بصريًا، هتُفتح مرحلة منفصلة لبناء الداشبورد (إدارة المحتوى، حجز الاستشارات + الدفع، إدارة المشاريع العقارية، إدارة الوظائف)، والتي هتعتمد بالكامل على بنية `/content/` والـ Design Tokens الجاهزة من هذا البريف دون أي إعادة بناء للفرونت إند.
